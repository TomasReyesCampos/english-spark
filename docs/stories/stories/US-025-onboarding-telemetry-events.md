## Contexto

Cada story del onboarding (US-010 a US-024) emite telemetry events
individuales. Esta story consolida la **infraestructura** de
telemetry para que sean consistentes:
- Schema unificado.
- PostHog setup.
- Funnel correctamente configurado para análisis.
- Property naming conventions (snake_case).

Sin este setup, cada story tendría su propia versión y los datos
serían inconsistentes para análisis.

UX implications en
[`onboarding-flow.md`](../../product/onboarding-flow.md) §18 (lista
de 17 events).

## Scope

### In

- Setup de PostHog SDK en cliente (`posthog-react-native`).
- Wrapper `track(eventName, properties)` con:
  - Auto-injection de `user_id` (si logged), `device_id`,
    `app_version`, `platform`.
  - Validation de event name (snake_case, verbo en pasado).
  - Logging local (Sentry breadcrumbs).
- Definición autoritativa de los 17 events del onboarding como

  - Step 1: `auth.<provider>.success`

## Acceptance criteria

- **Given** user en cualquier pantalla del onboarding, **When**
  emite un event, **Then** PostHog recibe el event con
  `user_id`, `device_id`, `platform`, `app_version` auto-inyectados.
- **Given** dev intenta `track('OnboardingStarted', {})`, **When**
  validation runs, **Then** retorna error console "Event name must
  be snake_case".
- **Given** user emite `onboarding.location_selected` con
  `country: 'MX'`, **When** PostHog recibe, **Then** la property
  `country` está visible en el event.
- **Given** funnel configurado en PostHog, **When** se analiza,
  **Then** completion rates por step son visibles (target ≥ 85%
  cada uno).
- **Given** PostHog está down, **When** se intenta emit, **Then**
  events se queue local (max 100) y se flush cuando vuelva
  conectividad.
- **Given** un test runs, **When** call `track`, **Then** mock
  PostHog recibe el call con properties exactas (verifiable en
  test).
- **Given** event `onboarding.completed` emitido, **When** llega a
  PostHog, **Then** payload incluye `active_track`, `cefr_estimate`,
  `primary_goals`, `total_time_minutes` (calculado del primer
  evento al último).
- **Given** Anonymous user, **When** emite events sin `user_id`,
  **Then** events se trackean con `anonymous_id` de PostHog
  (luego se mergean al hacer signup vía `posthog.identify`).

### Implementación esperada

```typescript
// app/lib/telemetry.ts
import PostHog from 'posthog-react-native';
import { ONBOARDING_EVENTS } from './event-names';

const ALL_EVENT_NAMES = [...ONBOARDING_EVENTS, /* otros sets */] as const;
type EventName = typeof ALL_EVENT_NAMES[number];

const posthog = new PostHog(Config.POSTHOG_API_KEY, {
  host: 'https://app.posthog.com',
  flushAt: 10,
  flushInterval: 10000,
});

export function track<T extends EventName>(
  event: T,
  properties: Record<string, any> = {}
) {
  // Validate snake_case
  if (!/^[a-z_]+\.[a-z_]+$/.test(event)) {
    console.error(`Invalid event name: ${event}. Must be snake_case dotted.`);
    return;
  }

  posthog.capture(event, {
    ...properties,
    platform: Platform.OS,
    app_version: Application.nativeApplicationVersion,
  });

  // Sentry breadcrumb
  Sentry.addBreadcrumb({
    category: 'telemetry',
    message: event,
    data: properties,
    level: 'info',
  });
}

export function identify(userId: string, traits: Record<string, any> = {}) {
  posthog.identify(userId, traits);
}
```
